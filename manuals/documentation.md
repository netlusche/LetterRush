## 1. Overview

LetterRush is a **local multiplayer word-sorting game** ‚Äî one device, passed between players. It turns *Stadt-Land-Fluss* into a Tinder-style swipe race: each player gets a timed turn sorting a deck of word cards by whether they match a rotating **(letter + category)** target.

The app is a pure **static single-page app**. There is **no backend and no external API** ‚Äî all ~2,850 words are bundled as JSON, so the game is fully playable offline. It installs as a PWA on iOS and Android directly from the browser.

## 3. State Management

### GameState

Defined in `src/types/index.ts`:

```typescript
interface GameState {
  lang: Language;                 // 'en' | 'de'
  theme: Theme;                   // one of 15 theme keys
  players: Player[];
  currentPlayerIndex: number;
  currentRound: number;
  totalRounds: number;            // 1‚Äì5
  phase: GamePhase;
  selectedCategoryIds: string[];
  settings: GameSettings;         // { roundSeconds, timePenalty }
  round: RoundConfig | null;      // { phases: RoundPhase[] }
  pendingResults: PlayerRoundResult[];  // locked, revealed after last player
  history: RoundHistoryEntry[];
}
```

`GamePhase` = `'SETUP' | 'CATEGORY_SELECTION' | 'PASS_DEVICE' | 'SORTING' | 'SCORE_LOCK' | 'ROUND_REVEAL' | 'FINAL_RESULTS'`.

### Reducer

`src/state/gameReducer.ts` ‚Äî a pure function (it does call `buildRound`, which uses `Math.random`, when a round starts). Actions:

| Action | Trigger |
|---|---|
| `CONTINUE_TO_CATEGORIES` | Step 1 confirmed ‚Üí store players/rounds/timer/lang |
| `BACK_TO_SETUP` | "Back" from the category screen |
| `START_GAME` | Categories confirmed ‚Üí build round 1, go to PASS_DEVICE |
| `BEGIN_SORTING` | Player taps "Start" on the pass-device screen |
| `LOCK_SCORE` | Turn ends (deck empty or time up) ‚Üí store result, hide it |
| `CONTINUE_AFTER_LOCK` | Next player, or (after last player) apply scores + write history ‚Üí reveal |
| `NEXT_ROUND` | Build next round, or finish to FINAL_RESULTS |
| `PLAY_AGAIN` | Reset scores/history, rebuild round 1 |
| `SET_THEME` | Theme dropdown |
| `RESET_GAME` | "Start over" ‚Üí back to SETUP |

### Persistence

`GameContext.tsx` debounces (300 ms) and writes the full state to `localStorage` under `letterrush_state`; it is restored on load.

## 5. Quiz Mechanic: Rotating Letter + Category

A turn is **`PHASES_PER_ROUND` (4) √ó `CARDS_PER_PHASE` (5) = 20 cards** (constants in `poolGenerator.ts`).

- Each **phase** has a target = `{ letter, categoryId }`, e.g. *Tier ¬∑ M*.
- The **card shows only the word** ‚Äî the target lives in the banner above (`SortingScreen`), which animates and flashes **"Neue Vorgabe!"** when the target changes between phases.
- **Right swipe is correct only if** the word starts with the current letter **AND** belongs to the current category.
- Per phase, traps are mixed and biased toward the hard kind:
  - ~60 % *right letter, wrong category* (drawn from other categories) ‚Äî the tricky ones,
  - the rest *right category, wrong letter*, then *both wrong* as filler.

`belongs` is precomputed at generation time, so swipe evaluation is a simple comparison at play time.

## 6. Round Generation & Anti-Memorization

`src/utils/poolGenerator.ts`:

- `validLettersForCategory(categoryId)` ‚Äî letters with ‚â• `MIN_HITS_PER_LETTER` (5) words in that single category. (Per-phase targets use one category, so there is **no** cross-category letter-intersection bottleneck.)
- `buildRound(selectedCategoryIds)` ‚Äî picks `pool = selected categories that are usable`; if none, widens to the full registry. Then builds 4 phases, each choosing a category from the pool + a valid letter, avoiding two identical consecutive targets, and filling the phase with `buildPhaseCards`.
- `buildPhaseCards(letter, categoryId)` ‚Äî draws 2‚Äì3 random hits + traps from large shuffled pools; deduplicates by word within the phase.
- `shuffleDeckForPlayer(phases)` ‚Äî per player: shuffles phase order **and** cards within each phase, then flattens to `DeckCard[]` stamped with each card's target.

Because every round redraws random subsets and every player gets a fresh shuffle, repeated category+letter combinations look different each time.

## 7. Scoring

`src/utils/scoring.ts` ‚Äî `evaluateSwipe(belongs, guess, elapsedMs, streak)`:

| Rule | Value |
|---|---|
| Base (correct) | +10 |
| Speed bonus (`elapsedMs < 2000`) | +5 |
| Combo multiplier | streak ‚â•3 ‚Üí √ó1.2, ‚â•5 ‚Üí √ó1.5, ‚â•7 ‚Üí √ó2 |
| Wrong swipe | ‚àí5, streak resets to 0 |
| Time penalty (optional) | ‚àí2 s off the round timer per mistake |

Points = `round((10 + speed) √ó multiplier)`. `SortingScreen` tracks `score / correct / wrong / bestCombo / streak`, locks the result on finish, and shows a floating `+N` flash per swipe.

## 8. Categories & Word Data

See **`manuals/data.md`** for the full data system. In short:

- `src/data/categories.ts` is the registry: each entry has `id`, `tier` (`'core' | 'bonus'`), localized `label`, and `data` (imported JSON).
- Word lists live in `src/data/*.json` (`{ kategorie, eintraege: [{ wort, anfangsbuchstabe }] }`) and are **generated** by `scripts/generate-categories.mjs` ‚Äî never edited by hand.
- 11 core + 8 bonus categories, ~2,850 entries.

## 9. High Scores

`src/services/highscores.ts` ‚Äî a top-10 table in `localStorage` (`letterrush_highscores`).
- `recordScores(players)` merges all players' final scores, keeps the top 10, and returns which entries are newly listed.
- `WinnerScreen` records once on mount, highlights new entries with a "NEU/NEW" tag, and shows a **"New record!"** badge if the new #1 came from this game.
- `GameSetup` shows the list (collapsible) with a clear button.

## 11. Themes & Canvas Animations

- 15 themes as `.theme-*` CSS-variable blocks in `styles.css`. The default is the LetterRush petrol/orange logo palette; the original neon palette lives on as **"Neon Party"** (last in the picker).
- `BackgroundEffects.tsx` runs a single `<canvas>` animation loop, switching effect by theme (e.g. drifting orbs for Default & Neon Party, Matrix rain, embers, petals, bubbles). Honors `prefers-reduced-motion`.
- `confettiColors.ts` provides a matching confetti palette per theme.

## 12. Localization

`src/i18n/translations.ts` holds flat `de` and `en` objects. The active language is part of game state. Category labels are localized in `categories.ts`. The app defaults to German.

## 13. PWA

- `vite-plugin-pwa` (Workbox `generateSW`) precaches the app shell **and** all bundled JSON, so the game is fully offline.
- `useWakeLock` keeps the screen awake during the game and re-acquires the lock when the tab returns to the foreground.
- Manifest name/colors set in `vite.config.ts`; icons in `public/` are generated from `branding/logo-source.png`.

## 14. Build, Versioning & Deployment

- `npm run build` runs `tsc` then `vite build` ‚Üí static `dist/`.
- **Versioning**: the start-screen footer shows `v{version}` imported from `package.json`; Vite inlines the value at build time, so every build reflects the current `package.json` version. To release, bump `"version"` in `package.json`, update the docs (this file, README, data.md) as needed, then build.
- **Hosting**: any static host works (GitHub Pages, Netlify, Vercel, Strato, IONOS). `vercel.json` provides an SPA rewrite. `base: './'` makes every asset/manifest/SW path relative, so the app also runs from a **subdirectory** (e.g. `example.com/letterrush/`). Note: `og:image`/`twitter:image` are relative too; for best social-share previews set them to an absolute URL once the final domain is known.

## 15. Known Design Decisions & Constraints

- **No backend / no API** ‚Äî all data is bundled; the game is offline-first by design.
- **`buildRound` uses `Math.random`** inside the reducer when a round starts. Pragmatic and matches the reference projects; not a pure reducer in the strict sense.
- **Card categories are hidden during play** ‚Äî players must judge category membership themselves; only the target banner names the category. This is what makes the category dimension meaningful.
- **Bonus categories are intentionally smaller / uneven** ‚Äî they are opt-in "expert" categories.
- **Round score can go negative** on many wrong swipes (no clamp), matching the spec's ‚àí5 penalty.
- **Preview sandbox quirk**: in the in-browser preview tool, background timers are throttled, so the countdown can look frozen; it runs normally in a real foreground browser.
